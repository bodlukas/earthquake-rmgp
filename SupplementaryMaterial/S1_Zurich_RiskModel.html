Ground motion module¶

This module builds upon an empirical ground-motion model (GMM) and an empirical spatial correlation model to simulate random fields of a ground-motion intensity measure (IM) conditional on a scenario earthquake with characteristics $\mathbf{e}$. The GMM provides the mean function and the standard deviation of the between-event and within-event residual. The spatial correlation model provides the lengthscale used for the covariance function of the within-event residual. Here we use the GMM of Akkar and Bommer [1] and the spatial correlation model of Esposito and Iervolino [2].

[1] Akkar, S. and Bommer J.J. (2010): Empirical Equations for the Prediction of PGA, PGV, and Spectral Accelerations in Europe, the Mediterranean Region, and the Middle East; Seismological Research Letters, 81(2):195-206; 10.1785/gssrl.81.2.195

[2] Esposito, S. and Iervolino, I. (2011): PGA and PGV Spatial Correlation Models Based on European Multievent Datasets; Bulletin of the Seismological Society of America, 101(5):25322541; 10.1785/0120110117

Covariance function¶

In [6]:
corr_length = 13.5 # Estimated from European data by Esposito and Iervolino (2011)

# Correlation function of within-event residuals:
def corr_fun_deltaW(df_target, corr_length):
    X = X2 = df_target[GMM.column_names[:2]].values
    # Matrix with Site-to-site distances
    DistMatrix = np.sqrt( np.sum( 
        np.stack([(X[:,dim].reshape(-1,1) - X2[:,dim].reshape(-1,1).T)**2 for dim in [0, 1] ]), 
        axis=0) )
    return np.exp(-3 * DistMatrix / corr_length)

# The covariance function
def cov_fun_gm(df_target, corr_length, T):
    stdB, stdW = GMM.get_sigmas(T)
    CorrMatrix = corr_fun_deltaW(df_target, corr_length)
    return stdW**2 * CorrMatrix + stdB**2

Damage module¶

First, load the vulnerability functions in a dataframe ('df_frag') and arrange the threshold parameters in matrix $\boldsymbol{\eta}$ ('etas') and the dispersion parameters in vector $\boldsymbol{\beta}$ ('betas'). These parameters are specified for 18 different classes $b\in\mathcal{B}=\{1,2,...,18\}$, namely unreinforced masonry structures with flexible floors ('URM_ff') and stiff floors ('URM_sf'), reinforced concrete shear wall buildings ('RC_SW') and reinforced concrete infill frame buildings ('RC_INF'). The RC structures are further distinguished by whether they have been built prior to the existance of modern seismic provisions in building codes ('preC') and after the existance of such provisions ('postC'). All of them are then further separated into different height classes: low-rise (LR: less than 3 stories) mid-rise (MR: between 3 and 5 stories) and high-rise (HR: more than 5 stories). For URM buildings we consider identical vulnerability functions for mid- and high-rise buildings.

The initially chosen functions employ 4+1 damage states, whereas in the current model we consider 3+1 damage states $y\in\mathcal{Y}=\{0,1,2,3\}$. Specifically, we lump the damage states 2 and 3 used by the vulnerability functions to $y=2$. The original damage state 4 becomes then $y=3$.

Typological attribution module¶

The present model categorizes residential buildings in the case-study region into four unkown typological attribute combinations $a\in\mathcal{A}=\{1,2,3,4\}$:

  • $a=1$ : Unreinforced masonry with flexible floors ('URM_ff')
  • $a=2$ : Unreinforced masonry with stiff floors ('URM_sf')
  • $a=3$ : Reinforced concrete shear wall ('RC_SW')
  • $a=4$ : Reinforced concrete infill frame ('RC_INF')

These attribute combinations are modelled via four latent functions $\mathbf{g}$ that follow independent gaussian processes with an identical covariance function but different mean functions. Conditional on realizations from these four latent functions, the class membership probabilities $p(A=a|\mathbf{g},\mathbf{X})$ are calculated via the softmax function. Conditional on the sampled attribute combinations and the input information, we then map to the vulnerability types from above using a deterministic function. The following paragraphs walk you through the different parts of the model starting with the mean function.

Mean functions¶

The mean functions of the latent functions $\mathbf{g}$ are based on an expert-based attribution model that comes in the form of a conditional probability table. Specifically, the herein assumed model provides class membership probabilities conditional on five construction periods. The table is shown below.

Out[11]:
ConPeriod p_URM_ff p_URM_sf p_RC_SW p_RC_INF
0 cy $\leq$ 1900 1.00 0.00 0.00 0.0
1 1900 $<$ cy $\leq$ 1944 0.60 0.40 0.00 0.0
2 1944 $<$ cy $\leq$ 1969 0.25 0.60 0.15 0.0
3 1969 $<$ cy $\leq$ 1988 0.00 0.25 0.45 0.3
4 cy $>$ 1988 0.00 0.10 0.60 0.3

We employ a smoothed version of this table via the introduction of sigmoid terms at the discrete boundaries of the different construction periods. The code below shows the formula and the figure illustrates the resulting functions.

# Add a small value to zero entries and normalize such that the rows sum up to one
AttMat = df_am.iloc[:, 1:].values

def mean_prior_prob_ta(df_target, AttMat):
    ca = AttMat.shape[1]
    Xm = np.zeros((len(df_target), ca))
    yb = df_target['const_year'].values
    for a in np.arange(ca):
        Xm[:, a] = (
            AttMat[0,a] + 
            (AttMat[1,a]-AttMat[0,a]) * 1/(1+np.exp(-0.5*(yb-1900))) +
            (AttMat[2,a]-AttMat[1,a]) * 1/(1+np.exp(-0.5*(yb-1944))) +
            (AttMat[3,a]-AttMat[2,a]) * 1/(1+np.exp(-0.5*(yb-1969))) +
            (AttMat[4,a]-AttMat[3,a]) * 1/(1+np.exp(-0.5*(yb-1988))) )
    return Xm

The mean functions $\mathbf{u}(\mathbf{x})$ of the four gaussian processes $\mathbf{g}$ are then the logarithmic probabilities shown in the above figure.

Covariance function¶

The covariance function of the gaussian process is a combination of four individual functions. The first one ('v_ls') uses the spatial distance with a lengthscale of 5 km to account for long-scale spatial correlations. The second one ('v_ss') is the multiplication of three functions, using spatial distance (but with a shorter length-scale of 0.15 km) and similarites with respect to construction year and number of stories. The short-scale covariance function ('v_ss') adds additional correlation for buildings that are close, as well as of similar age and height.

In [15]:
def k_sq(X1, X2, variance, lengthscale, dims):
    # Matrix of the squared euclidean distance between X1 and X2 in the specified dimensions (dims)
    sq_dist = np.sum( 
        np.stack( [ (X1[:,dim].reshape(-1,1) - X2[:,dim].reshape(-1,1).T)**2 for dim in dims ] ), 
        axis=0)
    return variance * np.exp(- sq_dist / (2*lengthscale**2))

def cov_fun_ta(dftarget):
    X1 = X2 = dftarget[['x', 'y', 'const_year', 'num_stories']].values
    v_ls = k_sq(X1, X2, 0.8, 5, [0, 1])
    v_ss = ( k_sq(X1, X2, 0.6, 0.15, [0, 1]) *
             k_sq(X1, X2, 0.6, 5, [2]) *
             k_sq(X1, X2, 0.6, 5, [3]) )
    return v_ls + v_ss

Deterministic mapping to vulnerability types¶

Mapping from the four unknown typological attribute combinations, $\mathcal{A}$, and the inputs, $\mathcal{X}$, to the 18 vulnerability types, $\mathcal{B}$, using the deterministic function $h$: $$h: \mathcal{A}\cup\mathcal{X} \mapsto \mathcal{B}$$ Specifically, we use the construction year to indicate whether a RC buildings was built prior or after the introduction of modern seismic provisions in building codes and the number of stories to categorize a building into the three height classes.

In [16]:
def AssignVulClass(sim_a, df_target):
    num_story = df_target['num_stories'].values
    con_year = df_target['const_year'].values
    sim_b = 3 * sim_a
    # Separate height classes according to number of stories
    sim_b += (num_story<=2) * 0 + ((num_story>2) & (num_story<=5)) * 1 + (num_story>5) * 2
    # For RC buildings: Separate pre- and post-code buildings according to the construction year and
    # the introduction of modern seismic provisions in building codes.
    sim_b += ((sim_a >= 2) * (con_year > 1988)) * 6
    return sim_b.astype('int')

We illustrate the typological attribution model on following toy example consisting of five buildings. As shown below, buildings 0, 1 and 3 are close to each other, whereas their distance to buildings 2 and 4 is 2km and 10km respectively. Note also the similarities and differences in the construction year ('GBAUJ') and the number of stories.

First, evaluate the mean functions $\mathbf{u}(\mathbf{x})$ of the typological distribution model evaluated on the inputs of those five buildings. The left plot below shows in each column the mean vector $\mathbf{u}^{(a)}$ for the different typological attribute combinations $a\in\mathcal{A}$. The right plot shows the exponentiated mean vectros, which correspond to the average class probabilities shown in the Figure above. Because these class probabilities only depend on the construction year, values for buildings 0, 1 and 4 are identical (i.e. 60\% probability for URM_ff).

Whereas the mean functions of the four GPs differ, their covariance function $v(\mathbf{x},\mathbf{x}')$ is the same. We illustrate first the covariance matrix resulting from an evaluation of the large-scale covariance function ($v_{ls}$) at the input points of the five buildings. Because building 4 is further away from the remaining buildings, its correlation (i.e. the off-diagonal terms) are small.

The following figure shows the covariance matrices resulting from an evaluation of the small-scale covariance functions ($v_{ss,1}$, $v_{ss,2}$, $v_{ss,3}$) at the input points of the five buildings.

First, $v_{ss,1}$ assigns high correlation (the off-diagonal terms) within the three buildings that are close to each other. Second, $v_{ss,2}$ assigns decreasing correlations with increasing difference in the construction year. Third, $v_{ss,3}$ assigns decreasing correlations with increasing difference in the the number of stories.

The multiplicative structure of the short-scale covariance function yields a covariance matrix where only buildings 0 and 1 are partially correlated and the others are uncorrelated. This follows from an element-wise multiplication of three matrices illustrated above.

Finally, the combined covariance matrix $v=v_{ls}+v_{ss}$ is shown below. We observe particularly high off-diagonal terms for the first two buildings. For building 4, which is located roughly 10km away from the others, the correlation is small.

The four latent functions evaluated on the input points of the five buildings result in four multivariate normal distributions, which have the same covariance matrix but different mean vectors. Now, we generate one sample from each of these distributions. This is illustrated in the left plot below. We see that especially for the highly correlated buildings 0 and 1 the sampled function values are very similar.

Subsequently, we apply the softmax function to derive class-membership probabilities: $$ p(A_i=a|\mathbf{g}_i) = \frac{\exp g_i^{(a)}}{\sum_j \exp g_i^{(j)}} $$ The resulting sampled class membership prob. are shown in the right plot.

These sampled class membership probabilities are then used to sample an attribute combination $a\in\mathcal{A}$ for each building independently, and subsequently allocate a vulnerability type $b\in\mathcal{B}$ using the deterministic function above. Error metrics¶

We use the simulated building counts from above to illustrate the error metrics used in the paper. Following "true" building counts per damage state $\check{\mathbf{n}} = [65, 25, 8, 2]$ are assumed.

Note that here we only performed 10 simulation runs, whereas for the results presented in the paper we used 10'000 runs.

Marginal Prediction Error (MPE)¶

The marginal prediction error, as defined in the paper, is calculated as the continuous ranked probability score (CRPS). For a specific damage state $y$, we evaluate the marginal prediction error as: $$ \mathrm{MPE}_{y}\hat{=}\mathrm{CRPS}(p(N_{y});\check{n}_{y}) \approx \frac{1}{r} \sum_{j=1}^{r} \left( || n_{yj} - \check{n}_{y} ||_2 - \frac{1}{2r} \sum_{i=1}^{r} || n_{yj} - n_{yi} ||_2 \right) $$

The first term within the sum of the MPE formulation measures the bias of the predictions. Note that the euclidean distance in one dimension is equivalent to the absolute difference. Thus, for each simulation run $j$, the first term measures the absolute error with respect to the true number of buildings in that damage category, $\check{n}_{y}$. This is shown in the first row of the following plot for damage state $y=1$.

In [34]:
true_vals = np.array([65, 25, 8, 2])
term_1 = np.abs(sam_n[1,:] - true_vals[1])

The second term of the MPE accounts for the dispersion. For each simulation run $j$ it measures the average absolute difference to all simulation runs, which is then multiplied by 0.5. This is shown in the second row of the following plot for damage state $y=1$.

In [35]:
term_2 = np.stack([1/(2*numSim) * np.sum(np.abs(sam_n[1,i] - sam_n[1,:])) 
                   for i in np.arange(numSim)])

The MPE is then calculated via subtracting the second term from the first term, and taking the average of this result over all $r$ simulation runs.

Joint Prediction Error (JPE)¶

The joint prediction error, as defined in the paper, is calculated as the energy score (ES). The ES is the generalization of the continuous ranked probability score to multivariate distributions and is defined as: $$ \mathrm{JPE} \ \hat{=} \ \mathrm{ES}(p(\mathbf{N});\check{\mathbf{n}}) \approx \frac{1}{r} \sum_{j=1}^{r} \left( || \mathbf{n}_{j} - \check{\mathbf{n}} ||_2 - \frac{1}{2r} \sum_{i=1}^{r} || \mathbf{n}_{j} - \mathbf{n}_{i} ||_2 \right) $$ This definition closely resembles the definition of the MPE, except that the scalar values $n_{y,j}$ and $\check{n}_{y}$ are replaced by vectors $\mathbf{n}_{j}$ and $\check{\mathbf{n}}$.

The first term within the sum sign measures the euclidean distance between the simulated damage state counts $\mathbf{n}_{j}$ in run $j$ and the true damage state counts $\check{\mathbf{n}}$. The euclidean distance is measured in a four-dimensional space, since we consider four damage states.

The second term measures the average euclidean distance from the simulated damage state counts $\mathbf{n}_{j}$ to the damage state counts of all simulation runs, which is then multiplied by 0.5.

Both terms are illustrated in the figure below for each simulation run.
