Risk model¶

Ground motion module | Damage module | Typological attribution module

In contrast to the notebook S1 (Zurich Case Study), this notebook presents the prior risk model directly in its object-oriented implementation, used in the risk-model informed gaussian process (RMGP). The code of RMGP is publicly available.

Ground motion module¶

This module builds upon an empirical ground-motion model (GMM) and an empirical spatial correlation model to simulate random fields of a ground-motion intensity measure (IM) conditional on a scenario earthquake with characteristics $\mathbf{e}$. The GMM provides the mean function and the standard deviation of the between-event and within-event residual. The spatial correlation model provides the lengthscale used for the covariance function of the within-event residual. Here we use the GMM of Bindi et al. (2011) [1] and the spatial correlation model of Esposito and Iervolino (2011) [2].

The object SpatialGMM combines a ground-motion model and a spatial correlation model. This is the first module of the RMGP.

Damage module¶

First, load the vulnerability functions in a dataframe ('df_frag') and arrange the threshold parameters in matrix $\boldsymbol{\eta}$ ('etas') and the dispersion parameters in vector $\boldsymbol{\beta}$ ('betas'). The herein employed vulnerability functions have been published in [1] and [2], and are also part of the recently published national Italian seismic risk assessment [3].

The vulnerability function parameters are specified for 12 different classes $b\in\mathcal{B}=\{1,2,...,12\}$, where the first six pertain to three classes of unreinforced masonry structures with decreasing suspectibility towards earthquake-induced damage ('A', 'B', 'C1') that are further separated into low-rise (less than 3 stories) and mid-/high-rise buildings (at least 3 stories). The last six pertain to two classes of RC buildings built before ('C1') and after ('D') seismic classification of the region of interest, which are further separated into three height classes low-rise (LR: less than 3 stories) mid-rise (MR: 3 and 4 stories) and high-rise (HR: more than 4 stories). Notebook S2 provides further information on these classes.

The vulnerability functions employ 5+1 damage states (EMS-98), whereas in the case-study we consider 3+1 damage states $y\in\mathcal{Y}=\{0,1,2,3\}$. Specifically, damage states 2 and 3 used by the vulnerability functions are lumped to $y=2$. The damage states 4 and 5 then become damage state $y=3$.

The object DamageEstimationModule collects the vulnerability function parameters $\boldsymbol{\eta}$ and $\boldsymbol{\beta}$. It transforms these parameters into a GPflow likelihood object that is used for inference using inspection data. This is the second module of the RMGP.

Typological attribution module¶

The present model categorizes residential buildings in the case-study region into four unkown typological attribute combinations $a\in\mathcal{A}=\{1,2,3,4\}$:

  • $a=1$ : Unreinforced masonry of class A
  • $a=2$ : Unreinforced masonry of class B
  • $a=3$ : Unreinforced masonry of class C1
  • $a=4$ : Reinforced concrete

These attribute combinations are modelled via four latent functions $\mathbf{g}$ that follow independent gaussian processes with an identical covariance function but different mean functions. Conditional on realizations from these four latent functions, the class membership probabilities $p(A=a|\mathbf{g},\mathbf{X})$ are calculated via the softmax function. Conditional on the sampled attribute combinations and the input information, we then map to the vulnerability types from above using a deterministic function. The following paragraphs walk you through the different parts of the model starting with the mean function.

Mean functions¶

The mean functions of the latent functions $\mathbf{g}$ are based on an expert-based attribution model that comes in the form of a conditional probability table. Specifically, the herein assumed model provides class membership probabilities conditional on six construction periods and separates between low-rise buildings and mid-/high-rise buildings. The two tables are shown below.

Because we assume to know the construction period and not the exact construction year as in the Zurich case study (notebook S1), these conditional probability tables are not smoothed. The mean functions $\mathbf{u}(\mathbf{x})$ of the four gaussian processes $\mathbf{g}$ are the logarithmic probabilities stated in the aforementioned tables.

First combine both tables in a three-dimensional array and then define a function that attributes the mean values to each building based on its construction period and height class.

def mean_prior_prob_ta(df_target, AttMat):
    C_A = AttMat.shape[1]
    CP = df_target['ConstrPeriod'].values.codes
    HC = df_target['HeightClass'].values.codes
    Xm = np.zeros((len(df_target), C_A))
    for bc in np.arange(C_A):
        for cp in np.arange(6):
            for hc in np.arange(3):
                Xm[:, bc] += AttMat[cp, bc, hc] * ((CP==cp) & (HC==hc))
    return Xm

Covariance function¶

The covariance function of the gaussian process is a combination of four individual functions. The first one ('v_ls') uses the spatial distance with a lengthscale of 5 km to account for long-scale spatial correlations. The second one ('v_ss') is the multiplication of three functions, using spatial distance (but with a shorter length-scale of 0.15 km) and similarites with respect to construction period and height class.

Because the construction period and height class are discrete variables, we specify a linear kernel on these attributes. Specifically, this covariance function considers buildings as correlated if they have the same construction period (for 'v_ss_2') and the same height class (for 'v_ss_3'). The multiplicative structure of the short scale covariance function ('v_ss') models correlations only for geographically close buildings built in the same period and of the same height class.

The object TypologyAttributionModule collects the functions explained above and initializes GPflow compatible kernel (covariance function), mean function and likelihood objects. This is the third, and final, module of the RMGP. The deterministic mapping to vulnerability types is added in the next section.

Deterministic mapping to vulnerability types¶

Mapping from the four unknown typological attribute combinations, $\mathcal{A}$, and the inputs, $\mathcal{X}$, to the 18 vulnerability types, $\mathcal{B}$, using the deterministic function $h$: $$h: \mathcal{A}\cup\mathcal{X} \mapsto \mathcal{B}$$ Specifically, we use the construction period to indicate whether a RC buildings was built prior or after the introduction of modern seismic provisions in building codes and the number of stories to categorize a building into the three height classes.

def AssignVulClass(sim_a, df_target):
    height_class = df_target['HeightClass'].values.codes
    con_period = df_target['ConstrPeriod'].values.codes
    sim_b = 2 * sim_a
    # For URM buildings: Separate only low-rise and Mid-/High-rise buildings
    sim_b += (sim_a <= 2) * ((height_class == 0) * 0 + (height_class > 0) * 1)    
    # For RC buildings: Separate all height-classes
    sim_b += (sim_a == 3) * ((height_class == 0) * 0 + (height_class == 1) * 1 
                             + (height_class == 2) * 2)
    # For RC buildings: Separate pre- and post-code buildings according to the construction period and
    # the seismic classification of the region.    
    sim_b += ((sim_a == 3) * (con_period > 2)) * 3
    return sim_b.astype('int')

TAM.func_assign_vul_class = AssignVulClass
