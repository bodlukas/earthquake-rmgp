S2b: Pollino Case-study - Building database augmentation¶

This notebook describes the augmentation of the building database and imputation of missing information. Specifically, the data obtained via the damage database and pre-processed in notebook S2a does not cover the entire building stock present in the region of interest.

The data augmentation includes four steps:

  1. Calculation of the number of missing buildings in each municipality based on 2001 ISTAT census data.
  2. Assign unique geo-coordinates to each added building based on the European Settlement Map.
  3. Fill missing attributes (for example the construction period) based on 2001 ISTAT census data.
  4. Attribute a EuroCode8 soil class to all buildings based on Forte et al. (2019)

This notebook includes all steps and assumptions, such that interested persons can reproduce the augmentation by: (1) following notebook S2a and (2) downloading the raw census data, European Settlement Map and soil class map. As of January 2021, the access to the three data sources is public. The full references of the underlying data sources are stated in the main paper.

Pre-processing of census data¶

From the ISTAT2001 census data, following information for each municipality (comune) is used:

  • E3: Total number of residential buildings
  • E6: Number of masonry buildings
  • E7: Number of reinforced concrete buildings
  • E9: Number of buildings built before 1919
  • E10: Number of buildings built between 1919 and 1945
  • E11: Number of buildings built between 1946 and 1961
  • E12: Number of buildings built between 1962 and 1971
  • E13: Number of buildings built between 1972 and 1981
  • E14: Number of buildings built between 1982 and 1991
  • E15: Number of buildings built after 1992
  • E16: Number of buildings with one story
  • E17: Number of buildings with two stories
  • E18: Number of buildings with three stories
  • E19: Number of buildings with four and more stories

The variable 'PRO_COM' is used to identify the different municipalities.

Attribute remaining missing building information¶

First, attribute the construction period, the height class and the building material (RC or Masonry). This is done by comparing the total number of buildings in a given construction period with the number of inspected buildings in that construction period. This gives us the proportions of the remaining buildings, based on which we sample a construction period for each building.

Attribute Soil class¶

Forte et al. (2019) provide soil conditions, in terms of EuroCode8 soil classes, for Italy as maps together with a supporting software for data export. A map for the region of this case-study was exported and geo-referenced using standard GIS tools. The map indicates three different soil classes (A,B and C) with different colors. For each building, the soil class of the closest pixel in the map is attributed based on its color.

Forte G., Chioccarelli E., De Falcoa M., Cito P., Santoa A. and Iervolino I. (2019): Seismic soil classification of Italy based on surface geology and shear-wave velocity measurements; Soil Dynamics and Earthquake Engineering; 122: 79-93

The first step pertains to building a dataframe with all pixel centers and the corresponding soil class.

# This short function provides soil class of the closest pixel center for each building
from scipy.spatial import cKDTree
def ckdnearest(gdA, gdB):

    nA = np.array(list(gdA.geometry.apply(lambda x: (x.x, x.y))))
    nB = np.array(list(gdB.geometry.apply(lambda x: (x.x, x.y))))
    btree = cKDTree(nB)
    dist, idx = btree.query(nA, k=1)
    gdB_nearest = gdB.iloc[idx].drop(columns="geometry").reset_index(drop=True)
    gdf = pd.concat(
        [
            gdA.reset_index(drop=True),
            gdB_nearest['val']
        ], 
        axis=1)

    return gdf

# Apply the function
gdfTotSC = ckdnearest(gdfTot, d)
