Risk model¶

Ground motion module | Damage module | Typological attribution module

In contrast to the notebook S1 (Zurich Case Study), this notebook presents the prior risk model directly in its object-oriented implementation, used in the risk-model informed gaussian process (RMGP). The code of RMGP is publicly available.

Ground motion module¶

This module builds upon an empirical ground-motion model (GMM) and an empirical spatial correlation model to simulate random fields of a ground-motion intensity measure (IM) conditional on a scenario earthquake with characteristics $\mathbf{e}$. The GMM provides the mean function and the standard deviation of the between-event and within-event residual. The spatial correlation model provides the lengthscale used for the covariance function of the within-event residual. Here we use elastic, 5% damped spectral acceleration at T=0.3 seconds as IM, the GMM of Akkar and Bommer [1] and the spatial correlation model of Esposito and Iervolino [2].

The object SpatialGMM combines a ground-motion model and a spatial correlation model. This is the first module of the RMGP.

References

[1] Akkar, S. and Bommer J.J. (2010): Empirical Equations for the Prediction of PGA, PGV, and Spectral Accelerations in Europe, the Mediterranean Region, and the Middle East; Seismological Research Letters, 81(2):195-206; 10.1785/gssrl.81.2.195

[2] Esposito, S. and Iervolino, I. (2012): Spatial Correlation of Spectral Acceleration in European Data; Bulletin of the Seismological Society of America, 102(6):27812788; 10.1785/012012006

Damage module¶

First, load the vulnerability functions in a dataframe ('df_frag'). The herein employed vulnerability functions have been published in [1] and the different vulnerability types are chosen based on [2].

The vulnerability function parameters are specified for 12 different classes $b\in\mathcal{B}=\{1,2,...,12\}$, namely adobe structure ('ADO'), unreinforced masonry structures with two different brick technologies ('URM_A') and ('URM_B'), and confined masonry structures from three different construction periods ('MCF_A', 'MCF_B' and 'MCF_C'), where each type is further separated into low-rise (1-2 storeys) and mid-rise (3-4 storeys) buildings. Because the dataset is limited to single-family houses, there are no buildings with more than 4 storeys.

Then arrange the threshold parameters in matrix $\boldsymbol{\eta}$ ('etas') and the dispersion parameters in vector $\boldsymbol{\beta}$ ('betas'). The initially chosen functions employ 4+1 damage states, whereas in the current model we consider 3+1 damage states $y\in\mathcal{Y}=\{0,1,2,3\}$. Specifically, we lump the damage states 2 and 3 used by the vulnerability functions to $y=2$. The original damage state 4 becomes then $y=3$.

The object DamageEstimationModule collects the vulnerability function parameters $\boldsymbol{\eta}$ and $\boldsymbol{\beta}$. It transforms these parameters into a GPflow likelihood object that is used for inference using inspection data. This is the second module of the RMGP.

Typological attribution module¶

The present model categorizes residential buildings in the case-study region into four unkown typological attribute combinations $a\in\mathcal{A}=\{1,2,3,4\}$:

  • $a=1$ : Adobe ('ADO')
  • $a=2$ : Unreinforced masonry with old brick technology ('URM_A')
  • $a=3$ : Unreinforced masonry with new brick technology ('URM_B')
  • $a=4$ : (Partially) confined masonry ('MCF')

These attribute combinations are modelled via four latent functions $\mathbf{g}$ that follow independent gaussian processes with an identical covariance function but different mean functions. Conditional on realizations from these four latent functions, the class membership probabilities $p(A=a|\mathbf{g},\mathbf{X})$ are calculated via the softmax function. Conditional on the sampled attribute combinations and the input information, we then map to the vulnerability types from above using a deterministic function. The following paragraphs walk you through the different parts of the model starting with the mean function.

Mean functions¶

The mean functions of the latent functions $\mathbf{g}$ are based on an expert-based attribution model that comes in the form of a conditional probability table. Specifically, the herein assumed model provides class membership probabilities conditional on six construction periods. The table is shown below.

AttMat = df_am.iloc[:, 1:].values

def mean_prior_prob_ta(df_target, AttMat):
    C_A = AttMat.shape[1]
    Xm = np.zeros((len(df_target), C_A))
    yb = df_target['const_year'].values
    for bc in np.arange(C_A):
        Xm[:, bc] = (
            AttMat[0,bc] + (AttMat[1,bc]-AttMat[0,bc]) * 1/(1+np.exp(-0.5*(yb-1933))) +
            (AttMat[2,bc]-AttMat[1,bc]) * 1/(1+np.exp(-0.5*(yb-1950))) +
            (AttMat[3,bc]-AttMat[2,bc]) * 1/(1+np.exp(-0.5*(yb-1963))) +
            (AttMat[4,bc]-AttMat[3,bc]) * 1/(1+np.exp(-0.5*(yb-1975))) +
            (AttMat[5,bc]-AttMat[4,bc]) * 1/(1+np.exp(-0.5*(yb-1990))) )
    return Xm

Deterministic mapping to vulnerability types¶

Mapping from the four unknown typological attribute combinations, $\mathcal{A}$, and the inputs, $\mathcal{X}$, to the 12 vulnerability types, $\mathcal{B}$, using the deterministic function $h$: $$h: \mathcal{A}\cup\mathcal{X} \mapsto \mathcal{B}$$ Specifically, we use the construction year to indicate whether a MCF buildings is of type A, B or C.

In [38]:
def AssignVulClass(sim_a, df_target):
    num_story = df_target['num_stories'].values
    con_year = df_target['const_year'].values
    sim_b = 2 * sim_a
    # Separate height classes according to number of stories
    sim_b += (num_story<=2) * 0 + ((num_story>2)) * 1
    # For MCF buildings: Separate buildings according to the construction year.    
    sim_b += (
        ((sim_a >= 2) * ((con_year > 1975) & (con_year <= 1990))) * 2 +
        ((sim_a >= 2) * (con_year > 1990)) * 4
        )
    return sim_b.astype('int')
TAM.func_assign_vul_class = AssignVulClass
